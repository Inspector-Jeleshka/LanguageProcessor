## Название и цель лабораторной работы

Лабораторная работа 4. Реализация алгоритма поиска подстрок с помощью регулярных выражений  
Цель: изучить теоретические основы регулярных выражений и их применение для поиска и извлечения
подстрок из текста, освоить практические навыки использования библиотечных средств работы
с регулярными выражениями, а также интеграцию алгоритмов поиска в графический интерфейс приложения.

Коченков А. А.

## Постановка задачи

Разработать модуль поиска подстрок с использованием регулярных выражений,
интегрировать его в существующее приложение (текстовый редактор) и обеспечить наглядный вывод результатов.

**Требования к программе:**
- Интегрировать модуль поиска в существующий интерфейс текстового редактора (ЛР1).
- Добавить в интерфейс элементы управления для выбора типа поиска (выпадающий список или кнопки) и запуска анализа.
- Результаты поиска должны отображаться в таблице с колонками:
  - Найденная подстрока (извлеченный фрагмент).
  - Начальная позиция (номер строки, номер символа).
  - Длина (количество символов).
- При выборе строки в таблице результатов соответствующая подстрока в области редактирования
  должна подсвечиваться (изменение цвета фона или выделение).
- Возможность вывода количества найденных совпадений.

**Входные и выходные данные:**
- Вход: текст, введенный пользователем в область редактирования.
- Выход: таблица найденных подстрок с указанием позиций, длины и количества совпадений;
  подсветка соответствующих фрагментов в исходном тексте.

## Решение задач

13\. Построить РВ для того, чтобы найти все годы между 2000 и 2010.  
Регулярное выражение: `20(0\d|10)`  
Значение:
- 20 - литерал "20". Это общая часть для всех искомых годов, которые начинаются с "20".
- ( - начало группировки. Она нужна, чтобы объединить далее идущие символы.
- 0\d - вариант А:
  - 0 - символ "0".
  - \d - метасимвол, обозначающий любую цифру от 0 до 9.
  Вместе 0\d даёт: "0" + любая цифра - числа 00, 01, ..., 09.
- | - оператор альтернации. Разделяет варианты внутри группы.
- 10 - вариант Б: литерал "10".
- ) — закрывающая группировку скобка.

Примеры: 2000, 2001, 2010  
Антипримеры: 2011, 1999

21\. Построить РВ, описывающее положительные и отрицательные целые числа и
    числа с плавающей точкой (разделители запятая и точка).  
Регулярное выражение: `[+-]?\d+([\.,]\d+)?`  
Значение:
- [+-]? - квантификатор ? (0 или 1 раз) применяется к символьному классу [+-]:
  - [+-] - символьный класс, означающий один символ: либо +, либо -.
  - ? - делает знак необязательным (0 или 1 вхождение). Таким образом, число может быть без знака (положительное), со знаком + или со знаком -.
- \d+ - метасимвол \d (любая цифра от 0 до 9) с квантификатором + (один и более раз). Это целая часть числа.
- ( - открытие группировки для необязательной дробной части.
- [\\.,] - символьный класс, означающий один символ: либо точка \\. (экранирована, т.к. точка в РВ — спецсимвол), либо запятая ,. Это разделитель целой и дробной части.
- \d+ - один и более цифр (дробная часть).
- ) - закрытие группы.
- ? - квантификатор для всей группы - означает, что дробная часть может отсутствовать. То есть число может быть целым (без дробной части) или с дробной частью.

Примеры: 0, 123, +42, -999, 3.14, 123,45, 0.5  
Антипримеры: .5, -.5

11\. Построить РВ, описывающее путь к файлу в Windows.  
Регулярное выражение: `([A-Z]:|\.|\.\.)?(\\?(\w[\w ]+\w|\w+))+\.\w+`  
Значение:  
Часть 1 - необязательный корень пути
- ( - начало первой группы (корень/начало пути).
- [A-Z]: - буква диска от A до Z (заглавная), за которой следует двоеточие. Например: C:, D:.
- | - альтернация.
- \\. - экранированная точка - . (текущая папка).
- | - альтернация.
- \\.\\. — две экранированные точки - .. (родительская папка).
- ) - конец первой группы.
- ? - квантификатор ?. То есть корень пути необязателен (может быть C:, ., .. или вообще отсутствовать).

Часть 2 - одна или более папок (или имя файла без папок)
- ( - начало второй группы (одна папка/сегмент пути).
- \\\\? - обратный слеш \\ (экранирован, т.к. спецсимвол) с квантификатором ?.
  То есть перед именем папки может быть обратный слеш, а может не быть (для первого сегмента).
- ( - начало внутренней группы для имени папки.
- \w[\w ]+\w - вариант А (имя папки с пробелами внутри):
  - \w - любая буква, цифра или знак подчёркивания (первый символ).
  - [\w ]+ - символьный класс: буква/цифра/подчёркивание или пробел (один и более раз).
  - \w - последний символ обязательно буква/цифра/подчёркивание (не пробел).
  Это позволяет имена типа "My Documents", "Program Files", но не " folder " с пробелами по краям.
- | - альтернатива.
- \w+ - вариант Б: просто одно и более букв/цифр/подчёркиваний (без пробелов). Например: Windows, Users.
- ) - конец внутренней группы.
- ) - конец второй группы (один сегмент пути).
- \+ - квантификатор для второй группы (один и более раз). То есть как минимум один сегмент (папка или имя файла без расширения).

Часть 3 - расширение файла
- \\. - экранированная точка (разделитель перед расширением).
- \w+ - один и более букв/цифр/подчёркиваний (расширение файла). Например: txt, exe, docx, py.

Примеры: C:\Documents\Newsletters\Summer2018.pdf, \Custom Utilities\StringFinder.exe, January.xlsx,
..\Publications\TravelBrochure.pdf, C:\Projects\apilibrary\apilibrary.sln, C:Projects\apilibrary\apilibrary.sln  
Антипримеры: C:\folder, file, C:\my_file.
